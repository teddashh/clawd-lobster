# Webapp Scaffold

## Directory Structure

```
<workspace>/
├── CLAUDE.md
├── workspace.json
├── .gitignore
├── .dockerignore
├── knowledge/
├── skills/learned/
├── openspec/
│   ├── project.md
│   ├── changes/
│   └── specs/
├── src/
│   ├── app/                 # Application code
│   ├── static/              # Static assets (CSS, JS, images)
│   └── templates/           # HTML templates (if server-rendered)
├── tests/
│   ├── unit/
│   ├── integration/
│   └── e2e/
├── deploy/
│   ├── infra-spec.md
│   ├── Dockerfile
│   ├── docker-compose.dev.yml
│   ├── docker-compose.staging.yml
│   ├── docker-compose.prod.yml
│   ├── nginx/
│   │   ├── dev.conf
│   │   ├── staging.conf
│   │   └── prod.conf
│   ├── .env.example
│   └── scripts/
│       ├── deploy-staging.sh
│       └── deploy-prod.sh
├── .env.dev                 # gitignored
├── .env.staging             # gitignored
└── .env.prod                # gitignored
```

## Key Directories

- `src/app/` — Main application code. Structure depends on framework.
- `deploy/` — All Docker and deployment configs. Generated by `/deploy:build`.
- `openspec/` — Specs, proposals, designs. Generated by `/spec`.
- `tests/` — Unit, integration, and E2E tests.

## Notes

- `deploy/` is generated by `/deploy:build`, not during initial scaffold.
  During scaffold, only create the empty `deploy/` directory.
- `.env.*` files are always gitignored.
- `src/` structure adapts to the framework (e.g., Next.js uses `app/` or `pages/`).
