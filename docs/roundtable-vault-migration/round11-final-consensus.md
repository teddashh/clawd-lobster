### Views

```sql
-- View 1: Current (non-retracted, latest version) documents
CREATE OR REPLACE VIEW v_current_documents AS
SELECT * FROM vault_documents 
WHERE is_latest = 1 AND redacted_at IS NULL;

-- View 2: Document with latest version ID (for following version chains)
CREATE OR REPLACE VIEW v_document_with_latest AS
SELECT d.*, 
       CASE WHEN d.is_latest = 1 THEN d.id
            ELSE (SELECT MAX(d2.id) FROM vault_documents d2 
                  WHERE d2.is_latest = 1 
                  START WITH d2.id = d.id 
                  CONNECT BY PRIOR d2.id = d2.parent_doc_id)
       END AS latest_version_id
FROM vault_documents d;
```

## vault_api Methods (Final List)

| Method | Purpose | Input | Output |
|--------|---------|-------|--------|
| `ingest(doc)` | Insert document + source | PreDocument or dict | doc_id |
| `bulk_ingest(docs)` | Batch insert | List[PreDocument] | List[doc_id] |
| `search(query, filters)` | Hybrid search | query string + metadata filters | List[doc] |
| `entity_resolve(name)` | Find entity by any alias | name string | entity + aliases |
| `about(entity)` | Everything about an entity | entity name | docs + facts + relations |
| `timeline(entity, start, end)` | Chronological events | entity + date range | List[doc] |
| `relations(entity)` | Graph neighbors | entity name | List[relation] |
| `accept(doc_id)` | Promote lifecycle | doc_id | success/fail |
| `supersede(old_id, new_id)` | Version replacement | two doc_ids | success/fail |
| `retract(doc_id)` | Soft delete | doc_id | success/fail |
| `add_entity(name, type)` | Create entity | name + type | entity_id |
| `add_relation(src, tgt, type)` | Create graph edge | source + target + type | relation_id |
| `add_fact(entity, s, p, o)` | Create typed fact | entity + SPO triple | fact_id |
| `merge_entities(src, tgt)` | Merge two entities | two entity_ids | success/fail |
| `split_entity(eid, alias_ids)` | Split entity | entity_id + alias list | new_entity_id |
| `gdpr_erase(entity_id)` | Privacy deletion | entity_id | erase report |
| `absorb(source, type, platform)` | Universal ingestion | source path/url + hints | job summary |

## Design Principles (Unanimous Agreement)

1. **Everything is a document.** All content enters through vault_documents with doc_type discrimination. No per-type tables.

2. **The Vault stores knowledge, not files.** Content is text (or extracted-to-text). Original binaries stay in their original storage. The Vault is a search index and knowledge graph, not a file manager.

3. **Schema is stable, metadata is flexible.** Typed columns for things that need indexes (email_from, occurred_at, ownership). Everything else in metadata_json. New data types never require ALTER TABLE.

4. **Enrichment is separate from ingestion.** Migration/ingest is fast (INSERT only). Chunking, embedding, entity extraction, fact extraction happen asynchronously.

5. **Dedup is idempotent.** content_hash prevents exact duplicates. original_path matching creates versions. Re-running absorb on the same data is safe.

6. **Lifecycle governs visibility.** Agents see `extracted`/`accepted` by default. `noise` and `retracted` are hidden. `raw` is processing backlog.

7. **The graph layer is the killer feature.** vault_entities + vault_relations + vault_facts turn a flat document store into a knowledge graph. "Show me everything about David" works across all data types.

8. **Conservative entity resolution.** Don't auto-merge uncertain entities. Keep them separate, merge later with human confirmation. Wrong merges are harder to fix than missed merges.

9. **Privacy by design.** ownership + privacy_level on every document. GDPR erase is a first-class operation. Oracle ATP provides encryption at rest.

10. **Measure everything.** vault_metrics + vault_audit_trail provide quantified self-awareness. The Vault knows how big it is, how fast it is, and who's using it.
